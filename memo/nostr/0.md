# Nostr アドレスの作り方

## Nostr アドレスとは？

プロフィールURLを短縮したメールアドレス風URL。

https://nostter.app/profile

にある Nostr アドレス は 本来ならば以下のようなクソ長いURLを`ユーザ名@example.com`のようなメアド風の短い文字列で表してくれる奴。


## Nostr アドレスの作り方

1. HEXを作る()
2. GitHubの`ユーザ名.github.io`リポジトリに`/.well-known/nostr.json`を作る
2. 

### 1. HEXを作る

HEXとは公開鍵などから算出する鍵。

* NIP-19:bech32-encoded entities,BIP-173 / BIP-350 (Bitcoin Improvement Proposals)
* https://github.com/nostr-protocol/nips/blob/master/19.md
* [Nostrの鍵とHEXを変換][]

[Nostrの鍵とHEXを変換]:https://mugen-tools.com/tools/nostrkey.php
[Nostter]:https://nostter.app/

1. [Nostter][]にアクセスする
    1. `プロフィール`→`…`→`ID(npub)をコピー`
1. [Nostrの鍵とHEXを変換][]にアクセスする
    1. コピーした`npub...`を`変換元の鍵`にペーストする
    2. `HEXに変換する`ボタンを押す
    3. `結果`に出力されたテキストをコピーする

私の場合は以下だった。

```
npub1p8pfq8taltlpwqhjcgh98uze6gyfqkwnh5jxtfpakzhl6scefyes0klzh8
```
```
09c2901d7dfafe1702f2c22e53f059d2089059d3bd2465a43db0affd43194933
```

### 2. GitHubの`ユーザ名.github.io`リポジトリに`/.well-known/nostr.json`を作る

1. GitHubで`ユーザ名.github.io`リポジトリを作る（私の場合は`ytyaru.github.io`）
2. ルート直下で`_config.yml`ファイルを作成し内容を`include: [".well-known"]`とする
3. ルート直下で`.well-known`ディレクトリを作成する
4. 2の下に`nostr.json`ファイルを作る。内容は以下。

```json
{
  "names": {
    "希望する名前": "さっき取得したHEX"
  }
}
```

私の場合は以下。

```json
{
  "names": {
    "ytyaru": "09c2901d7dfafe1702f2c22e53f059d2089059d3bd2465a43db0affd43194933"
  }
}
```

1. `https://ytyaru.github.io/.well-known/nostr.json`にアクセスできることを確認する

尚、デプロイまでには数分ほど時間がかかる可能性がある。リポジトリの`Setting`→`Pages`から状態確認すること。デプロイ中だったらエラーっぽい表示が出ているはず。

URLは以下のようになるはず。

https://ytyaru.github.io/.well-known/nostr.json
